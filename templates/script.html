Network Reconnaissance Suite
Scripts
This script verifies the validity and security of an SSL certificate installed on a website or server. It ensures that the certificate is properly configured and up to date, providing encryption and authentication for secure connections.
This script scans for vulnerabilities that could allow attackers to inject malicious scripts into web pages viewed by other users. It helps identify and mitigate XSS vulnerabilities, which can lead to unauthorized access, data theft, and other security risks.
This script examines the website for potential clickjacking vulnerabilities, where attackers can trick users into clicking on unintended elements or buttons, leading to unauthorized actions or information disclosure.
This script checks for insecure mixed content on web pages, where secure (HTTPS) pages include non-secure (HTTP) resources such as images, scripts, or stylesheets. Such mixed content can compromise the security of the entire page and leave it vulnerable to various attacks.
This script evaluates whether the website implements HSTS, a security policy mechanism that helps protect websites against man-in-the-middle attacks and cookie hijacking by forcing secure connections (HTTPS).
This script detects vulnerabilities that could allow attackers to inject and execute malicious scripts in web applications, which are then reflected back to users' browsers, potentially leading to account compromise or data theft.
This script assesses whether the website's cache control headers are properly configured to prevent caching sensitive or private information, reducing the risk of unauthorized access to cached data.
This script investigates the information exposed by the web server, such as version numbers, server software, and configuration details. It helps identify potential security risks associated with disclosing too much information to attackers.
This script examines the website for vulnerabilities related to cache poisoning, where attackers manipulate cached content to serve malicious data to users, potentially leading to various security threats such as session hijacking or data manipulation.
This script looks for misconfigurations in CORS policies, which could allow unauthorized access to resources hosted on a different origin (domain). Improper CORS settings can lead to data leakage or unauthorized data access between websites.
